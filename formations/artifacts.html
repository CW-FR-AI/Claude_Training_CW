Module pédagogique · Formation 04

Comprendre et utiliser
les artifacts de Claude.

Le bon réflexe quand vous démarrez un livrable, et la formation complète pour comprendre les mécaniques. Chat, Cowork, partage, mises à jour, mini-apps avec IA embarquée — tout ce qu'il faut pour produire des artifacts qui tiennent la route.

À consulter avant de démarrer un livrable

Quel type d'artifact pour mon livrable ?

Décrivez ce que vous voulez produire. Vous obtiendrez l'environnement à utiliser (Chat ou Cowork), le format d'artifact recommandé, l'approche de partage et la méthode pour ne pas perdre de temps.

① Type de livrable
Pourquoi ce module existe Les artifacts sont puissants mais mal utilisés font perdre du temps : on lance un Chat artifact alors qu'on voulait un fichier Word, on partage un lien sans réaliser que les pièces jointes du chat sont aussi exposées, on multiplie les versions au lieu de les éditer. Avant de démarrer un livrable, prenez 30 secondes ici. Pour comprendre les mécaniques sous-jacentes, allez voir l'onglet Formation complète.

— Chapitre 1

Le concept — qu'est-ce qu'un artifact ?

Un artifact, c'est un contenu autonome que Claude produit dans une fenêtre dédiée, séparée du fil de conversation. Pas un message dans le chat, mais un objet qui vit en parallèle, qu'on peut éditer, faire évoluer, partager, et réutiliser ailleurs.

Quand Claude crée un artifact

Claude bascule en mode artifact dès que ce qu'il produit remplit ces critères :

  • Substantiel et autonome — typiquement plus de 15 lignes, qui se tient sans le contexte de la conversation
  • Vous allez vouloir le modifier ou itérer dessus
  • Vous allez vouloir le réutiliser hors du chat — l'envoyer, l'imprimer, le poster ailleurs
  • Vous allez vouloir y revenir plus tard

En pratique : un mémo de 800 mots, un tableau comparatif, un schéma, une mini-app, un fichier Word généré pour un client — tous ces livrables sortent en artifact. Une réponse courte, une explication conversationnelle, une réponse à une question factuelle — ça reste inline.

Fil de conversation
« Voici un comparatif des trois cibles que tu m'as demandé de regarder. Je l'ai mis dans l'artifact à droite — tu peux l'éditer ou me demander de le modifier. »
Fenêtre artifact
[ Tableau comparatif interactif — Cibles A, B, C × CA, EBITDA, croissance, fit ]

Chat ou Cowork — deux mécaniques très différentes

Le terme « artifact » désigne en réalité deux choses presque sans rapport selon l'environnement :

  • Dans le Chat (claude.ai web et desktop) — l'artifact est un objet rendu en direct dans le navigateur. HTML, React, Markdown, code. Il vit dans Claude, pas sur votre machine. On peut le partager via lien, l'embarquer, lui ajouter du persistent storage ou des appels à des MCP.
  • Dans Cowork (app desktop uniquement) — les « artifacts » sont en réalité des fichiers locaux générés sur votre disque dur, affichés dans un volet de prévisualisation. Ils vivent dans le dossier que vous avez donné en accès. Un fichier Excel généré par Cowork est un fichier Excel, point. Le volet Artifacts vous permet juste de le prévisualiser sans changer d'app. Depuis avril 2026, une nouveauté importante : les Live Artifacts, qui se rafraîchissent avec des données externes via les MCP.
Ne pas confondre Un artifact Chat partagé via lien reste sur les serveurs Claude — l'accès dépend du mode de partage choisi (voir Chapitre 4). Un fichier généré par Cowork reste sur votre disque dur, vous le partagez comme n'importe quel fichier. Ce sont deux régimes complètement différents en termes de sécurité et de partage.

L'inline, c'est le mail. L'artifact, c'est la pièce jointe vivante.

Un mail bien écrit suffit pour transmettre une info ponctuelle, une réponse à une question, une réflexion. Mais quand le contenu est destiné à être édité, partagé, retravaillé, on l'isole en pièce jointe — un Word, un Excel, une pièce qui peut circuler indépendamment du fil de discussion.

Les artifacts jouent exactement ce rôle, en mieux : ils sont vivants. On peut demander à Claude de les modifier, ils gardent leur historique de versions, on peut les partager avec un lien, ils peuvent même être interactifs ou appeler de l'IA pour répondre à leurs utilisateurs. Pensez en livrable plutôt qu'en réponse. Si ce que vous demandez à Claude est destiné à exister hors de la conversation, ça doit être un artifact.

— Chapitre 2

Maîtriser les artifacts du Chat

Sept formats disponibles, chacun avec son terrain de jeu. Choisir le bon dès le départ vous évite les itérations inutiles.

Les sept formats à connaître

FormatQuand l'utiliser
Markdown.mdNotes, mémos, rapports formatés, documents structurés. Le format par défaut pour de la prose qui doit rester lisible et copiable.
HTML.htmlPages web autonomes : profil société interactif, formulaire, landing page, mini-site. Tout le CSS et JS dans un seul fichier.
React.jsxComposants interactifs avec état : calculateurs, dashboards, jeux, formulaires multi-étapes. Bibliothèques disponibles (Recharts, lucide-react, lodash, d3, Tailwind).
SVG.svgSchémas vectoriels précis : org chart, structure deal, illustration technique. Idéal quand vous voulez réimporter dans Illustrator ou PowerPoint.
Mermaid.mermaidFlowcharts, diagrammes de séquence, gantt simples. Syntaxe textuelle légère — parfait pour itérer rapidement sur un schéma de process.
Code.py / .js / autresScripts Python, JS, ou autres langages, à copier et exécuter ailleurs. Pour le code de plus de 20 lignes — en deçà, ça reste inline.
Office & PDF.docx, .pptx, .xlsx, .pdfLivrables téléchargeables au format Office — typiquement pour un client ou un usage hors Claude. Plus fiable via Cowork pour des fichiers structurés (voir Chapitre 3).

Création, édition, itération

Claude crée l'artifact automatiquement quand le contenu remplit les critères du Chapitre 1. Vous pouvez aussi forcer la création (« mets ça dans un artifact »).

  • Pour modifier un artifact existant : décrivez le changement dans le chat. « Ajoute une colonne EBITDA au tableau », « rends le bouton plus discret », « passe le titre en majuscules ». Claude met à jour l'artifact dans la même fenêtre.
  • Si l'artifact génère une erreur (typique sur React) : un bouton Try fixing with Claude apparaît près de l'erreur. Cliquez — Claude reprend automatiquement avec le contexte de l'erreur.
  • Édition de message antérieur : si vous éditez un de vos messages plus haut dans le chat, vous créez une nouvelle branche d'artifacts à partir de ce point. Les versions précédentes restent accessibles.

Versioning

Toutes les versions d'un artifact sont sauvegardées automatiquement dans le chat. En haut du panneau artifact, un dropdown vous permet de naviguer entre versions, voir une version antérieure, repartir d'une version pour une nouvelle itération. Important : c'est par chat — un artifact n'existe pas indépendamment de la conversation qui l'a créé.

Persistent storage — les artifacts qui retiennent des données

Disponible sur les plans Pro, Max, Team et Enterprise. Un artifact peut stocker jusqu'à 20 Mo de données par artifact, qui persistent entre sessions et entre utilisateurs. Deux modes :

  • Personnel — chaque utilisateur garde ses propres données. Cas d'usage : un journal, un tracker personnel, des préférences.
  • Partagé — tous les utilisateurs voient et écrivent dans les mêmes données. Cas d'usage : un leaderboard, un suivi collectif, un wiki léger.

Pour notre équipe, ça ouvre des cas d'usage concrets : un tracker de pipeline deal partagé, un suivi de Q&A diligence consolidé, un journal de décisions sur un dossier — tout ça vivant dans un artifact, sans backend ni base de données.

Connexions MCP — les artifacts qui parlent à des apps

Disponible sur les plans Pro, Max, Team et Enterprise. Un artifact peut appeler des MCP servers (Microsoft 365, Google Drive, Asana, ou nos connecteurs Pappers, Grata, S&P Global) pour lire ou écrire des données réelles. Chaque utilisateur s'authentifie indépendamment sur ses propres connecteurs — ce qui veut dire qu'un artifact partagé en équipe utilise les comptes de chacun, pas du créateur.

À garder en tête Un artifact connecté à un MCP n'a pas accès aux données : c'est l'utilisateur qui ouvre l'artifact qui l'autorise (ou non) à lire son inbox, son drive, etc. Ça reste dans les rails de sécurité standard.
— Chapitre 3

Maîtriser les artifacts en Cowork

Le mot « artifact » est trompeur en Cowork. Ce ne sont pas des objets vivants comme dans le Chat — ce sont des fichiers locaux sur votre machine, affichés dans un volet de prévisualisation pratique.

Le bon modèle mental

Quand Cowork génère un Excel, un PowerPoint, un PDF ou n'importe quel fichier, il l'écrit dans le dossier que vous lui avez donné en accès. C'est un fichier comme un autre dans votre Finder ou Explorateur Windows. Le volet Artifacts de Cowork vous permet de le prévisualiser directement dans l'app, sans avoir à ouvrir Excel ou PowerPoint à chaque vérification — c'est tout.

Conséquence pratique : pour partager un fichier généré par Cowork, vous le partagez comme n'importe quel fichier — par mail, par Drive, en glissant dans Teams. Pas de lien public, pas de mécanique propre à Claude.

Workflow type

  1. Vous décrivez la tâche à Claude dans Cowork
  2. Claude planifie les étapes et exécute (lecture des fichiers d'entrée, traitement, génération du fichier de sortie)
  3. Le ou les fichiers générés apparaissent dans le volet Artifacts et dans votre dossier local
  4. Vous les ouvrez, les vérifiez, demandez des ajustements à Claude — qui réécrit le fichier
  5. Le fichier final est dans le dossier, prêt à être partagé

Pour un Excel multi-feuilles, un PowerPoint complet, un PDF formaté — Cowork est bien plus fiable qu'un artifact Chat, parce qu'il utilise nos skills internes (skills xlsx, pptx, docx, pdf) qui produisent des fichiers de qualité professionnelle.

Live Artifacts — la nouveauté de 2026

Depuis avril 2026, Cowork propose une fonctionnalité distincte appelée Live Artifacts, accessible depuis l'onglet du même nom dans la sidebar. Ce sont des artifacts qui se rafraîchissent automatiquement avec les données de vos apps connectées (Microsoft 365, Drive, MCPs) à chaque ouverture.

Cas d'usage : un dashboard de suivi de deal qui pulle automatiquement les emails récents et les meetings du calendrier, un tracker de mandats qui se met à jour à partir d'un Excel partagé. Vous le créez une fois — il reste à jour tout seul.

Limites importantes des Live Artifacts (1) Ils vivent localement sur la machine — ils ne se synchronisent pas si vous changez d'ordinateur. (2) Ils ne sont pas partageables encore (sur la roadmap). (3) Ils utilisent les connecteurs sans demander permission à chaque appel — accordez votre confiance avant de mettre des connecteurs sensibles dans un Live Artifact.

Limites de Cowork à connaître

  • Desktop uniquement — macOS et Windows, pas de version web ni mobile native. L'app desktop doit rester ouverte pendant l'exécution.
  • Disponible sur les plans Pro, Max, Team et Enterprise.
  • Pas de partage natif des sessions ni des artifacts — le partage se fait via les fichiers locaux générés.
  • Mémoire limitée aux Projects Cowork — en session standalone, Claude n'a pas de mémoire persistante d'une session à l'autre.
  • Mobile pour Pro/Max — possibilité de lancer une tâche depuis le mobile, à condition que l'app desktop reste ouverte côté machine.
— Chapitre 4

Partager et faire vivre un artifact

Trois mécaniques de partage très différentes selon où vous êtes et quel plan vous avez. À bien comprendre, parce que chacune a des implications de sécurité différentes — particulièrement critiques sur des dossiers deal sensibles.

Mode 1 — Publish

Lien public

Plans individuels (Pro, Max)

Vous générez un lien public. N'importe qui avec le lien peut voir et interagir avec l'artifact, sans compte Claude. Code source visible. Embeddable sur un site externe. Pour des artifacts non confidentiels uniquement. Non disponible sur Team/Enterprise — chez Clearwater, vous utiliserez plutôt le Mode 2.

Mode 2 — Share

Partage organisation

Plans Team et Enterprise

Le partage se restreint aux membres de l'organisation Clearwater. Les viewers doivent être loggés sur leur compte Team/Enterprise pour accéder. Si l'artifact est dans un Project, ils doivent aussi avoir accès au Project. C'est le mode pertinent pour la plupart de nos livrables internes.

Mode 3 — Cowork

Fichier local

Tous plans avec Cowork

Pas de partage natif. Vous partagez le fichier comme n'importe quel fichier local — par mail, par Drive, par Teams. C'est le mode le plus simple pour des livrables clients (Word, Excel, PowerPoint).

Avertissement critique pour les chats avec pièces jointes Quand vous partagez un artifact créé dans un chat, les viewers gagnent aussi accès aux pièces jointes et fichiers présents dans la conversation qui l'a créé. Si vous avez chargé un CIM, un term sheet ou des comptes pour analyser une cible, et que vous partagez ensuite un artifact issu de ce chat, le destinataire peut accéder à ces documents. À vérifier systématiquement avant tout partage d'artifact issu d'un chat avec des docs deal sensibles. Solution : faire l'analyse sensible dans un chat, puis recréer l'artifact dans un chat propre sans les pièces jointes.

Mises à jour après partage

Lorsqu'un artifact est partagé, le lien pointe vers une version précise au moment du partage. Si vous modifiez l'artifact ensuite, vous créez une nouvelle version. Le destinataire continue de voir la version partagée — sauf si vous re-partagez explicitement la nouvelle version. À l'inverse, vous pouvez à tout moment révoquer le partage (bouton Unshare).

Embed sur un site externe

Pour les artifacts publiés (Mode 1), un bouton Get embed code génère un snippet HTML à coller sur un site. Il faut configurer les allowed domains côté Claude. Cas d'usage potentiel pour Clearwater : intégrer un mini-outil ou un calculateur sur un éventuel microsite, sur Notion interne, etc. Le Mode 2 (Team/Enterprise) ne propose pas l'embed pour des raisons évidentes de sécurité.

— Chapitre 5

AI-powered artifacts — l'IA embarquée

La fonctionnalité la moins connue mais la plus puissante. Un artifact peut appeler Claude lui-même pendant son exécution, devenant une mini-application intelligente — sans clé API, sans backend, sans coût additionnel pour le créateur.

Le principe

Vous demandez à Claude de coder un artifact qui, à un moment, appelle l'API Claude pour analyser, résumer, classer, ou répondre à une question. Le code écrit par Claude embarque cet appel. Quand un autre utilisateur ouvre l'artifact :

  • Il s'authentifie automatiquement avec son propre compte Claude
  • L'appel à l'IA tourne sur l'infra Anthropic — pas de clé API à gérer
  • L'usage compte sur son propre quota, pas sur celui du créateur
  • Pour Team/Enterprise, c'est sans coût additionnel en usage interne (pas de surcoût pour le créateur, l'usage rentre dans les abonnements de chacun)

Cas d'usage concrets pour l'IB

Cas 1

Analyseur de term sheet

L'utilisateur colle un term sheet en texte brut. L'artifact appelle Claude pour extraire les clauses clés (prix, conditions suspensives, garanties, earn-out, lock-up) et les structure dans un tableau standardisé. Réutilisable par toute l'équipe.

Cas 2

Mini-screener de cibles

L'utilisateur saisit une description de cible idéale (secteur, taille, géo, signaux M&A). L'artifact appelle Claude pour générer une short-list candidate, scorée et justifiée. Idéal pour les premières passes rapides en amont d'un mandat.

Cas 3

Polish de pitch text au format Clearwater

L'utilisateur colle un brouillon de tagline ou de bullet de slide. L'artifact appelle Claude (avec les règles Clearwater encodées dans le prompt) pour le réécrire au format attendu. Démocratise les standards d'écriture sans avoir à former chacun individuellement.

Cas 4

Q&A sur un document de référence

L'utilisateur charge un document (CIM, charte interne, guide). L'artifact appelle Claude pour répondre à des questions sur le contenu. Avec persistent storage, l'historique des questions reste pour les utilisateurs suivants.

Quand utiliser ça vs un workflow API direct

Un AI-powered artifact est parfait pour des outils distribués à l'équipe avec un usage ponctuel (quelques fois par semaine, par utilisateur). Pour des workflows à très haut volume, à très basse latence, ou qui demandent du caching de prompt sophistiqué, mieux vaut passer par l'API directe — avec les leviers de coût discutés dans la formation tokens.

Le bon test Si l'outil est appelé moins de 50 fois par jour au total dans l'équipe, et que la latence n'est pas critique : AI-powered artifact. Au-delà, ou si vous voulez intégrer l'outil dans un workflow plus large (Cowork, script), passez par l'API directe.
— Chapitre 6

Cas d'usage IB & bonnes pratiques

Synthèse des cas où les artifacts apportent une vraie valeur, et des pièges à éviter pour ne pas perdre du temps ou exposer ce qui ne doit pas l'être.

Six cas d'usage IB qui marchent particulièrement bien

Use case 1

Tableaux comparatifs interactifs

Peers, cibles, deals. Format React avec tri et filtres, ou Markdown si statique. Pour l'équipe, le format React permet de manipuler les données live sans rouvrir Excel.

Use case 2

One-pagers profil société

Format HTML autonome, qu'on peut ouvrir dans n'importe quel navigateur, qu'on peut imprimer en PDF, qu'on peut partager via lien interne. Plus rapide à itérer qu'un PowerPoint.

Use case 3

Dashboards de suivi de mandat

Format React avec persistent storage. Pipeline deal, suivi Q&A, capacité équipe. Avec connexion MCP à Microsoft 365, les données restent à jour sans intervention manuelle.

Use case 4

Mini-outils intelligents

Calculateurs (LBO simple, multiple implicite), analyseurs (term sheet, comptes annuels), formatters (pitch text, profil). Format React + AI-powered. Distribués à l'équipe via Share.

Use case 5

Schémas et structures de deal

Format SVG ou Mermaid. Pour les structures de financement, org charts post-deal, chronologies de process. Plus rapide à itérer qu'un schéma PowerPoint, exportable vers PowerPoint pour le livrable final.

Use case 6

Livrables Office pour client

Format Office via Cowork (skills xlsx, pptx, docx, pdf). C'est ici que Cowork brille — pour produire un Word formaté, un Excel multi-feuilles, un PowerPoint Clearwater, mieux vaut Cowork qu'un artifact Chat.

Six pièges classiques à éviter

1.

Demander un artifact pour 5 lignes

Si la réponse tient dans un message, gardez-la inline. Les artifacts pour des contenus courts saturent la fenêtre, polluent l'historique du chat, et n'apportent rien.

2.

Itérer sans fin dans le chat au lieu d'éditer le prompt

Si la première version d'un artifact rate la cible, ne dites pas « non, plutôt comme ceci… ». Éditez votre prompt initial, soyez plus précis, et regénérez. L'artifact sera plus propre et le contexte plus léger. (Voir formation tokens, Chapitre 5.)

3.

Multiplier les artifacts dans un même chat

Un chat avec 8 artifacts différents est lourd à charger, lent à itérer, et difficile à retrouver. Idéalement : un artifact principal par chat. Si vous avez besoin d'un autre artifact, ouvrez un nouveau chat.

4.

Partager sans vérifier les pièces jointes du chat

Le partage d'un artifact issu d'un chat expose les pièces jointes du chat. Avant de partager : vérifier ce qui a été chargé. Si docs sensibles, recréer l'artifact dans un chat propre.

5.

Confondre Chat artifacts et Cowork outputs

Un Excel généré dans un Chat artifact n'aura pas la même qualité qu'un Excel généré par Cowork (qui utilise les skills internes). Pour des fichiers Office structurés à destination client : Cowork.

6.

Oublier que les artifacts mobile ne s'éditent pas pareil

Sur l'app mobile, vous pouvez voir les artifacts et demander des modifications, mais l'expérience est dégradée par rapport au web/desktop. Les artifacts complexes (React interactifs, dashboards) gagnent à être créés et édités sur ordinateur.

Hygiène à généraliser dans l'équipe

  • Nommer ses artifacts dès la création — sinon ils se retrouvent dans la sidebar avec des noms génériques difficiles à retrouver.
  • Stocker les artifacts récurrents dans un Projet — un dashboard de suivi, un calculateur partagé, un formatter — pour qu'ils soient retrouvables et partageables au sein de l'équipe.
  • Réutiliser plutôt que recréer — un artifact existant qui marche peut être pointé vers de nouvelles données ou adapté à un nouveau besoin. Repartir de zéro à chaque fois est inefficace.
  • Documenter les artifacts importants — un AI-powered artifact distribué à l'équipe mérite une note d'utilisation (que fait-il, qui l'utilise, comment l'appeler).
Un artifact est un livrable, pas un brouillon. Il mérite la même rigueur que ce qu'on déposerait dans un dossier deal — parce que c'est exactement ce qu'il deviendra.
Annexes pratiques

Fiches mémo

Pensées pour être imprimées ou affichées à côté de l'écran.

FICHE 01

Quel format d'artifact pour quoi ?

  • Markdown — note, mémo, rapport texte structuré
  • HTML — page autonome, profil société, formulaire, mini-site
  • React — interactif avec état : calculateur, dashboard, jeu
  • SVG — schéma vectoriel précis, exportable PowerPoint
  • Mermaid — flowchart, séquence, diagramme rapide
  • Code — script Python, JS de plus de 20 lignes
  • Office (Cowork) — Word, Excel, PowerPoint, PDF formatés
Si c'est destiné à un client en format Office, passez par Cowork.
FICHE 02

Chat vs Cowork — qui partage quoi ?

  • Chat artifact — vit sur les serveurs Claude, partage via lien Share (Team/Enterprise)
  • Cowork output — fichier local sur votre disque, partage manuel par mail / Drive / Teams
  • Live Artifact (Cowork) — local uniquement, pas partageable encore
  • AI-powered artifact (Chat) — partageable, chacun consomme son propre quota Claude
Le partage natif n'existe que dans le Chat. Cowork, c'est du fichier local.
FICHE 03

Modes de partage

  • Publish (Pro, Max) — lien public, anyone peut voir, code source visible. Non disponible Team/Enterprise.
  • Share (Team, Enterprise) — restreint à l'organisation, viewers doivent être loggés. Le mode Clearwater par défaut.
  • Embed — disponible en mode Publish uniquement, avec configuration des allowed domains.
  • Unshare — possible à tout moment, désactive le lien.
Quand vous partagez, les pièces jointes du chat partent avec. Vérifier avant.
FICHE 04

Versioning

  • Toutes les versions d'un artifact sont sauvegardées automatiquement dans le chat
  • Dropdown en haut du panneau pour naviguer entre versions
  • Éditer un message antérieur crée une nouvelle branche d'artifacts
  • Un lien partagé pointe vers la version au moment du partage — modifier après ne met pas à jour le lien
  • L'historique de version vit dans le chat — pas d'export possible vers un autre chat
Si vous voulez réutiliser un artifact ailleurs : copier le code, ouvrir nouveau chat, repartir de là.
FICHE 05

AI-powered artifacts (Claudeception)

  • Artifact qui appelle Claude pendant son exécution
  • Pas de clé API à gérer, pas de backend à monter
  • Chaque utilisateur consomme son propre quota
  • Idéal pour outils distribués à l'équipe à usage modéré
  • Cas d'usage IB : analyseur term sheet, screener, polish pitch text, Q&A doc référence
Au-delà de 50 appels par jour cumulés dans l'équipe : passer par l'API directe.
FICHE 06

Pièges & anti-patterns

  • Un artifact pour 5 lignes → garder inline
  • Itérer en boucle dans le chat → éditer le prompt initial
  • Plus de 2-3 artifacts par chat → chat saturé, ouvrir nouveau
  • Partager sans vérifier les pièces jointes du chat
  • Fichier Office structuré dans le Chat → préférer Cowork
  • Mobile pour artifacts complexes → préférer desktop
L'artifact est un livrable. Il mérite la même rigueur qu'un dossier deal.
