Interactive To-Do.

Génère un dashboard React visuel de tâches, où vous pouvez marquer chaque item « done », « to-do » ou « removed », puis exporter vos choix vers Claude pour un suivi ou des actions complémentaires.

Type
Skill · Auto-trigger
Environnement
Chat (web, desktop)
Format de sortie
Artifact React (.jsx)
Modèle conseillé
Sonnet 4.6
— Ce qu'il fait

Une checklist vivante dans le chat

À partir d'une liste de tâches (que vous donnez en clair, ou qui résulte d'un Daily Email Review), le Skill produit un artifact React interactif : un dashboard où chaque tâche apparaît comme une carte cliquable. Vous cochez « done » au fur et à mesure, ou « removed » pour ce qui n'est plus pertinent.

Fonctionnalités du dashboard

  • Trois statuts par tâche — to-do, done, removed
  • Catégorisation par mandat / projet / contexte (héritée de la liste source)
  • Priorisation par criticité (urgent / important / routine)
  • Filtres par statut, par mandat, par priorité
  • Export de vos choix vers Claude pour un suivi (« voici ce que j'ai fait aujourd'hui »)

Persistance

L'artifact peut utiliser le persistent storage de Claude (voir formation Artifacts). Vos choix sont sauvegardés et vous pouvez retrouver votre to-do à la prochaine ouverture. Idéal pour une checklist qui se construit sur plusieurs jours.

— Déclenchement

Comment l'invoquer

Le Skill se déclenche automatiquement quand vous demandez une organisation de tâches ou une checklist interactive.

Formulations qui déclenchent

  • « Fais ma to-do »
  • « Organise mes tâches »
  • « Montre-moi ce que j'ai à faire »
  • « Interactive checklist »
  • « Task tracker »
  • « Quoi faire aujourd'hui »
  • « Priorise tout ça » — après un Daily Email Review
  • « Action items » à partir d'une liste donnée

Pattern le plus efficace : chaîné après Daily Email Review

  1. Lancez le Daily Email Review du matin (« lis mes emails depuis hier »)
  2. Une fois la synthèse produite, dites « maintenant priorise tout ça en checklist interactive »
  3. Le Skill produit l'artifact React, prêt à cocher au fur et à mesure
— Cas d'usage

Quand l'utiliser

Cas 1

To-do du jour après briefing email

Suite naturelle du Daily Email Review. Vous obtenez une vue claire de ce que vous devez faire, avec status en temps réel pendant la journée.

Cas 2

Checklist de kickoff mandat

À partir d'une checklist standard (NDA signé, IM brief, calendrier, univers initial, kickoff team), l'artifact React track l'avancement.

Cas 3

Suivi Q&A diligence

Pour un mandat en phase Q&A active, transformer la liste consolidée des questions en checklist suivie par statut (drafted, reviewed, sent).

Cas 4

Suivi d'un long process

Pour un mandat sur plusieurs mois, garder visible le statut de chaque livrable à produire — avec persistent storage, l'état persiste entre sessions.

— Pièges & limites

À garder en tête

Pas accessible depuis l'app mobile pour édition L'artifact React s'affiche correctement sur mobile mais l'interaction (cocher, drag, filter) est dégradée. Pour un usage régulier, utiliser le chat web ou desktop. Sur mobile, lecture seule recommandée.
  • Pas d'intégration native avec Outlook / Asana / Linear. L'artifact vit dans Claude. Pour synchroniser avec un outil externe, il faut le faire manuellement.
  • Persistence par utilisateur — vos choix dans l'artifact ne sont pas partagés avec votre équipe par défaut. Pour un tracker collectif, demander le mode « shared » dans le persistent storage.
  • Pour un suivi de pipeline complet (plusieurs mandats, plusieurs personnes, vraiment structuré), envisager plutôt un Project dédié avec une knowledge base — ou un outil externe comme Asana / monday.
