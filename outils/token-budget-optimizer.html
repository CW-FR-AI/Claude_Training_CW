Token Budget Optimizer.

Calibre Claude pour minimiser le gaspillage de tokens sur les tâches itératives — rédaction, révision, résumé, relecture. Réponses concises, pas de filler, scope serré sur la modification demandée plutôt que réécriture complète.

— Ce qu'il fait

Discipline d'écriture pour les tâches itératives

Le Skill change le comportement de Claude pour les tâches itératives (drafting, révision, résumé, polish). Au lieu de produire des réponses verbeuses qui réécrivent tout, Claude :

Comportements imposés

  • Clarifie le scope avant d'agir — si la demande est ambigüe, une question courte plutôt qu'une exécution dans le vide
  • Garde les réponses concises et professionnelles — pas de filler, pas de récapitulatif inutile
  • Pas de préambule — pas de « voici votre version corrigée », direct au contenu
  • Modifications scopées — édite uniquement le passage demandé ou le delta, sans réécrire l'ensemble
  • Multi-tour focalisé — pour des conversations itératives, garde le cap sur l'étape suivante au lieu de re-paraphraser

Pourquoi c'est utile

L'output coûte 5× l'input (voir formation Tokens). Pour une session de polish où vous itérez 10 fois sur un texte, économiser 50 % d'output à chaque tour = quota préservé, vitesse augmentée, et moins de pollution dans le contexte de la conversation.

— Déclenchement

Comment l'invoquer

Skill silencieuse. Se déclenche automatiquement sur les tâches itératives détectées.

Types de tâches qui activent le Skill

  • Rédaction (drafting)
  • Révision (revising)
  • Résumé (summarizing)
  • Relecture / correction (proofreading)
  • Polish
  • Itérations sur du texte existant

Et en français : rédaction, révision, résumé, relecture, correction, polish, itération, synthèse, reformulation.

Préciser pour un effet renforcé

Si vous voulez forcer la discipline encore plus, ajoutez explicitement :

  • « Réponse courte, pas de filler »
  • « Édite seulement le paragraphe X, ne touche pas au reste »
  • « Pas de préambule, donne-moi directement la version corrigée »
— Cas d'usage

Quand l'utiliser

Cas 1

Session de polish d'un mémo long

Vous itérez 10-15 fois sur des sections d'un mémo. Le Skill garde Claude focalisé sur les changements demandés au lieu de réécrire à chaque fois.

Cas 2

Drafting d'emails en série

Pour rédiger 10 emails first-touch en série, le Skill évite que Claude commente chaque mail produit — direct à l'output.

Cas 3

Résumé concis d'un long document

« Résume ce CIM en 10 points » — le Skill évite que la réponse fasse 1500 mots de paraphrase quand 200 suffisent.

Cas 4

Workflow Cowork récurrent

Sur un workflow Cowork facturé à l'API (plan de charge consolidé, batch de profils), la discipline du Skill réduit directement les coûts.

— Pièges & limites

À garder en tête

Ne pas l'utiliser pour de l'analyse de fond Pour des tâches qui demandent du raisonnement structuré (analyse de CIM, valorisation, stratégie), la concision imposée par le Skill peut faire perdre en qualité. Le Skill est calibré pour les tâches itératives, pas pour les analyses substantielles.
  • Pour un nouveau livrable from scratch, le Skill peut être trop sec. Précisez explicitement la longueur attendue si nécessaire (« 800 mots, factuel »).
  • Compatibilité avec les autres Skills. Combine bien avec Email Enhancer et Slide Text Enhancer (renforce la concision). Évite de combiner avec IB Prompt Enhancer (qui ajoute du structure et donc plus d'output).
  • Si la réponse vous semble trop courte, demandez explicitement plus de profondeur. Le Skill respecte les contraintes que vous donnez ensuite.
