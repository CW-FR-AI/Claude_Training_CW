— Ce qu'il fait

Analyse intégrée multi-modules

Le Skill orchestre une revue financière complète d'une cible. Il démarre par un intake interactif qui cadre le périmètre (entreprise, statut SaaS, modules pertinents, inputs disponibles) avant d'exécuter — et désactive les modules pour lesquels vous n'avez pas les données.

Les 10 modules disponibles

ModuleContenu
3-statement modelP&L, bilan, cash flow intégrés sur historique + projection 5 ans.
DCF valuationDiscounted cash flow avec WACC, terminal value, sensibilités.
LBO modelSources & uses, debt schedule, returns (MOIC, IRR), sortie multi-exit.
Accretion-dilutionImpact d'une acquisition sur le P&L de l'acquéreur (EPS impact).
Trading compsPeers cotés, multiples EV/EBITDA, EV/Sales, alignement comptable.
Transaction compsTransactions M&A comparables historiques.
Debt schedule & covenantsÉchéancier de dette, stress des covenants, capacité de servicing.
Sensibilités & scenariosMatrices de sensibilité (multiple, WACC, croissance), scénarios up/base/down.
Working capital & CCCCycle de cash conversion, jours de stocks/créances/dettes, retraitements.
Capital structure benchmarkingComparaison du gearing avec les peers, structure optimale.
SaaS metricsARR, NRR, churn, magic number, CAC payback — pour cibles software / recurring revenue.
— Déclenchement

Comment l'invoquer

Skill à invoquer dans Cowork (pas dans le chat web). Démarre toujours par un intake interactif.

Formulations qui déclenchent

  • « Revue financière de [société] »
  • « Valuation pack complet sur [cible] »
  • « Diligence model intégré »
  • « Revue M&A complète »
  • « Modèle d'évaluation pour [cible] »

L'intake interactif

Avant d'exécuter, le Skill vous pose des questions :

  1. Nom et description de la cible
  2. Cible SaaS / software / recurring revenue ? (active le module SaaS metrics)
  3. Quels modules voulez-vous activer ? (sélection à la carte)
  4. Inputs disponibles : historique financier, BP, hypothèses macro
  5. Format de sortie attendu : Excel multi-feuilles, mémo, slides

Répondre précisément à l'intake économise des allers-retours ensuite. C'est là que se joue la qualité de l'output.

— Cas d'usage

Quand l'utiliser

Cas 1

Diligence approfondie d'une cible

Sur mandat buy-side ou sell-side avec phase de modélisation. L'intake permet de cadrer modules selon les data disponibles.

Cas 2

Préparation d'IC interne

Slide ou mémo de valuation à présenter en IC. Demander en sortie un mémo synthétique avec annexes Excel.

Cas 3

Analyse d'une cible SaaS

Le module SaaS metrics dédié (ARR, NRR, churn, magic number) calibre l'analyse sur les bons KPIs du secteur.

Cas 4

Stress test d'une LBO structure

Module LBO + debt schedule + covenants pour valider la robustesse d'une structure de financement proposée.

— Pièges & limites

À garder en tête

Toujours vérifier les calculs avant un livrable client Le Skill produit un modèle structuré mais peut commettre des erreurs sur des formules complexes (boucles de calcul, sensibilités). Avant tout usage client, faire un audit du modèle — vérifier que le bilan ferme, que le cash flow se reconcilie, que les sensibilités sont cohérentes.
  • Cowork desktop nécessaire. Pas accessible depuis le chat web. L'app desktop doit rester ouverte pendant les calculs.
  • Inputs en une seule fois. Évitez d'ajouter des données au compte-gouttes — préparez l'ensemble (historique financier, BP, hypothèses) avant le lancement.
  • Pour les sensibilités, demandez une matrice ciblée (ex. : multiple d'entrée × multiple de sortie) plutôt que tout l'espace possible. Sinon le modèle devient illisible.
  • Module SaaS uniquement si pertinent. Sur cible non-software, désactivez-le à l'intake pour gagner du temps.
